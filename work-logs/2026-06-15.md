### 第一阶段：需求分析与架构设计

**用户输入：**
> 帮我做一个视频关键帧标注工具，纯前端的，一个 HTML 文件就行。

**AI 响应：**
确认需求：
- 单 HTML 文件，无后端依赖
- 支持加载本地视频文件（MP4/WebM/OGG）
- 在视频时间轴上添加关键帧标记
- 每个关键帧可填写标题和描述

- 使用 `<video>` 标签实现视频播放
- 使用 Canvas API 实现视频截图
- 使用 `FileReader` / `URL.createObjectURL` 处理本地文件
- 数据结构：`{ version: 1, keyframes: [{ time, title, desc, screenshot }] }`

### 第三阶段：视频播放功能

**用户输入：**
> 视频加载支持拖拽和点击选择两种方式。

**AI 实现：**

1. **文件选择：**
   - 创建隐藏的 `<input type="file">` 元素
   - 按钮点击触发 `input.click()`
   - 限制文件类型：`video/mp4, video/webm, video/ogg`

2. **拖拽上传：**
   - `dragover` 事件：显示红色边框提示
   - `dragleave` 事件：移除边框
   - `drop` 事件：验证文件类型并加载

3. **播放控制：**
   - 播放/暂停切换（`togglePlay()`）
   - 前进/后退 5 秒（`skipTime(±5)`）
   - 音量滑块（`volumeSlider`）
   - 播放速度选择（0.25x ~ 2x）
   - 进度条点击跳转

4. **时间显示：**
   - 格式：`MM:SS.mmm`（精确到毫秒）
   - 实时更新当前时间和总时长

### 第四阶段：关键帧管理核心功能

**用户输入：**
> 关键帧要能添加、删除、编辑，时间轴上也要有标记。

**AI 实现：**

1. **添加关键帧（`addKeyframe()`）：**
   - 获取当前视频播放时间
   - 防重复检测（0.1 秒内视为同一位置）
   - 自动生成默认标题：`关键帧 MM:SS.mmm`
   - 按时间顺序插入数组
   - 添加后自动选中并跳转

2. **删除关键帧（`deleteSelected()` / `deleteKf(id)`）：**
   - 支持从编辑面板删除
   - 支持从列表项快捷删除按钮删除
   - 删除后隐藏编辑面板

3. **编辑关键帧（`saveEdit()`）：**
   - 标题输入框
   - 描述文本域
   - 保存后刷新列表显示

4. **关键帧导航：**
   - 上一个关键帧（`prevKeyframe()`）
   - 下一个关键帧（`nextKeyframe()`）
   - 到达边界时循环跳转

5. **时间轴标记：**
   - 使用绝对定位的 `<div>` 元素
   - 金色竖线标记，hover 时变亮变高
   - 点击标记直接选中对应关键帧

### 第六阶段：截图功能

**用户输入：**
> 截图功能很重要，要能截取当前帧，保存为 PNG。

**AI 实现：**

1. **截图原理（`takeScreenshot()`）：**
   - 创建 Canvas 元素，尺寸与视频一致
   - `drawImage(video, 0, 0)` 绘制当前帧
   - `toDataURL('image/png')` 导出为 Base64

2. **关联关键帧：**
   - 如果当前有选中的关键帧，自动关联截图
   - 列表项显示 📸 图标标识有截图

3. **截图弹窗（Modal）：**
   - 显示截图预览
   - 下载 PNG 按钮（创建 `<a>` 标签触发下载）
   - 复制到剪贴板按钮（`ClipboardItem` API）
   - 点击遮罩层关闭

4. **文件命名：**
   - 格式：`keyframe_MM-SS-mmm.png`

### 第七阶段：导入导出

**用户输入：**
> 标注数据要能导出 JSON，也能从 JSON 导入。

**AI 实现：**

1. **导出（`exportData()`）：**
   - 数据结构：`{ version: 1, keyframes: [...] }`
   - 每个关键帧包含：`time`, `title`, `desc`, `screenshot`
   - 使用 `Blob` + `URL.createObjectURL` 生成下载链接
   - 文件名：`keyframes_YYYY-MM-DD.json`

2. **导入：**
   - 创建隐藏的 `<input type="file" accept=".json">`
   - `FileReader.readAsText` 读取文件
   - `JSON.parse` 解析并验证结构
   - 重新生成 ID（`Date.now() + index`）避免冲突
   - 按时间排序后渲染
